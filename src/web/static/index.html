Image-to-3D Preview

Thin UI wrapper for the CLI preview pipeline. Upload an image, tune two knobs, and see the preview.

n_colors: liczba kolorow w palecie (mniej = prostszy wynik).
blend_depth: glebokosc mieszania na granicach regionow (wiecej = plynniejsze przejscia).
Calls POST /preview
  1. wybierz obraz
  2. ustaw parametry
  3. kliknij Preview (lub poczekaj na live preview)

UI generuje tylko preview (PNG). STL i colorplan sa w trybie bundle/CLI.

  • No file selected
  • File too large (>20MB)
  • Invalid image
  • Backend rejected request (400)
  • Network error / server not reachable

These are preview-only errors (PNG only). They do not change outputs or files. The UI does not retry or use fallbacks.

  • Idle: no active request; preview may be empty or previous.
  • Rendering: /preview request in progress; UI may abort a previous request if inputs change.
  • Done: last request succeeded; preview matches the request shown in Last updated.
  • Error: last request failed; the error message is shown in status.

Changing n_colors or blend_depth triggers live preview automatically after a short debounce.

Does:
  • Upload image and call POST /preview
  • Show request summary (fields included/omitted)
  • Show status (Idle/Rendering/Done/Error) + error message
  • Show preview image (PNG)
Does not:
  • Generate STL
  • Generate colorplan.txt
  • Run /bundle
  • Store sessions or history
  • Modify filament catalog / sequence / allowed filaments
  • Action: click Generate preview with no file. Expect: Error + "No file selected".
  • Action: select a small image, click Generate preview. Expect: Rendering... then Done + preview visible.
  • Action: set n_colors and wait. Expect: Rendering after debounce, then Done.
  • Action: set blend_depth and wait. Expect: Rendering after debounce, then Done.
  • Action: stop the server, click Preview. Expect: Error + Network error.
  • Action: change file/params, check Request summary. Expect: fields show current values.

Last updated is the end time of the latest request (Done or Error). Preview matches current inputs compares the preview to the current file and parameters.

Endpoint /preview
Method POST
Payload multipart/form-data
file None

Included fields file only
Omitted fields n_colors, blend_depth
Payload details
  • file: UploadFile (image/*), always required
  • n_colors: int, sent only when filled (empty = omitted)
  • blend_depth: float, sent only when filled (empty = omitted)

UI does not send any other fields (catalog/sequence/allowed) on this screen.

UI sends only the fields listed above.

The CLI remains the source of truth. This UI only wraps the existing /preview endpoint.
